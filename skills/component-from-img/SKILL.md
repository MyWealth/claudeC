# 根据设计图实现组件功能

## 适用场景

需要在现有 Vue 3 组件中，根据设计图或截图添加新的表单功能、UI组件或交互逻辑。

## 工作流程

### 第一步：分析需求

1. **分析图片内容**
   - 识别所有UI元素（表单控件、按钮、文本等）
   - 理解元素之间的层级关系
   - 确认交互逻辑（条件显示、联动关系等）3

2. **读取目标组件**
   - 读取目标组件文件
   - 了解现有代码结构、数据状态、API调用方式
   - 确认使用的UI组件库

3. **确认理解**
   - 向用户确认需求理解是否正确
   - 指出可能存在疑问的地方

---

### 第二步：制定计划

1. **列出需要修改的内容**
   - 数据状态（formValue 新字段）
   - 数据选项（下拉选项等）
   - 验证规则
   - 模板结构
   - 样式
   - API 调用和提交参数

2. **征求用户同意**
   - 展示分步执行计划
   - 等待用户确认后再开始执行

---

### 第三步：分步执行

**每执行一步前都要询问用户是否确认**

#### 常见执行步骤：

1. **扩展数据状态** - 在 formValue/ref 中添加新字段
2. **添加数据选项** - 添加下拉选项等状态管理
3. **添加验证规则** - 在 rules 中添加验证逻辑
4. **添加模板结构** - 在 template 中添加表单项
5. **添加样式** - 添加 CSS 样式
6. **添加 API 调用** - 添加数据获取方法
7. **更新提交参数** - 在保存/更新方法中添加新参数
8. **重置状态** - 在关闭/重置方法中添加新字段的重置

---

## 执行原则

1. **确认原则** - 每一步执行前都要询问用户确认
2. **小步迭代** - 将大功能拆分成小步骤，每步都是可独立验证的
3. **保持一致** - 遵循现有组件的代码风格和结构
4. **不破坏现有功能** - 只做增量修改，不改变已有逻辑

## 注意事项

- 如果用户对某步有异议，先理解用户意图，再调整计划
- 遇到不确定的地方，主动询问用户
- 完成后总结所有变更内容
